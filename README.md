# HomeSec — домашний контроль доступа в сеть

Система контроля интернет-доступа для домашней сети: веб-панель для управления
устройствами, пользователями и гостями, блокировки и расписания, которые
**нельзя обойти на устройстве** — весь контроль исполняется на уровне сети
(MikroTik + AdGuard Home).

## Почему это работает, когда родительский контроль на ПК не работает

Родительский контроль на ПК дети обходят: другой пользователь, live-USB,
переустановка, смена DNS. HomeSec действует иначе:

- **Единственный шлюз.** Huawei GPON работает в bridge-режиме, весь трафик
  идёт через MikroTik. Другого пути в интернет физически нет.
- **Принудительный DNS.** Любой DNS-запрос (даже к 8.8.8.8 вручную)
  перехватывается фаервольным правилом и заворачивается на AdGuard Home.
- **Блокировка обходов.** DoH/DoT (шифрованный DNS), сторонние VPN-порты
  блокируются для контролируемых групп.
- **Идентификация по устройству.** Каждое устройство привязано к статическому
  IP по MAC и состоит в группе (дети / взрослые / гости / неизвестные).
  Новое неизвестное устройство автоматически попадает в ограниченную группу.

## Топология

```
Интернет (GPON)
   │
Huawei HG8546M ── bridge-режим (только оптика → ethernet)
   │
MikroTik hAP ac2 ── ГЛАВНЫЙ РОУТЕР: firewall, DHCP, NAT, контроль
 ├── Switch 1G ──────── ПК, приставки (проводные)
 ├── Archer C6 (AP) ─── Wi-Fi покрытие
 └── Raspberry Pi 3B ── 192.168.88.2
       ├── AdGuard Home  (DNS-фильтр, safe search, блок-листы)
       └── HomeSec panel (FastAPI, порт 8000)
```

## Компоненты

| Компонент | Роль |
|---|---|
| `backend/` | Веб-панель (FastAPI + Jinja2 + htmx, SQLite) |
| `mikrotik/homesec-base.rsc` | Базовая конфигурация RouterOS: address-lists, принудительный DNS, блок DoH/DoT, гостевой Wi-Fi |
| `docs/` | Пошаговые инструкции: перевод Huawei в bridge, настройка MikroTik, установка на Raspberry Pi |
| `deploy/` | systemd-юнит и скрипт установки на Pi |

## Как устроен контроль

1. **Группы устройств** — списки `hs-kids`, `hs-guests`, `hs-blocked`,
   `hs-unknown` (address-list в RouterOS). Панель добавляет/убирает IP
   через RouterOS API.
2. **Блокировка интернета** — IP устройства попадает в `hs-blocked`,
   forward-правило дропает его трафик. Мгновенно, без перезагрузок.
3. **Расписания** — панель (APScheduler) в заданное время включает/выключает
   блокировку группы или устройства. Состояние восстанавливается после
   перезагрузки Pi (reconcile при старте).
4. **Категории (игры, YouTube, соцсети)** — через AdGuard Home
   «Blocked services» персонально по IP клиента + блок-листы.
5. **Ограничение скорости** — simple queues на MikroTik, настраивается
   из панели (например, гостям 10 Мбит/с).
6. **Гости** — отдельный SSID на MikroTik, изолированная подсеть
   192.168.90.0/24, лимит скорости, доступ только в интернет.

## Быстрый старт

Порядок настройки (подробно — в `docs/`):

1. `docs/01-network-setup.md` — Huawei в bridge, физическая коммутация.
2. `docs/02-mikrotik-setup.md` — интернет на MikroTik + импорт `homesec-base.rsc`.
3. `docs/03-raspberry-pi-setup.md` — AdGuard Home + панель HomeSec на Pi.

После установки панель доступна на `http://192.168.88.2:8000`
(логин задаётся при первом запуске через `.env`).

## Дорожная карта (фаза 2 — ИИ)

Точки расширения уже заложены (`backend/app/services/` изолирует все
интеграции, события пишутся в `event_log`):

- ИИ-отчёты и аномалии: анализ DNS-логов через Claude API
  («всплеск DoH-запросов — похоже на попытку обхода», «новое устройство ночью»).
- Правила на естественном языке: «заблокируй детям игры после 21:00».
- Классификация трафика и квоты по категориям (2 часа игр в день).
