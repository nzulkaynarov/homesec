# Шаг 1. Перестройка сети: Huawei в bridge, MikroTik — главный роутер

Цель: весь трафик дома проходит через MikroTik. Только тогда блокировки
невозможно обойти, подключившись «в обход».

## Целевая схема

```text
Оптика провайдера
   │
Huawei HG8546M ── режим BRIDGE (только преобразует GPON в Ethernet)
   │ LAN1
   │
MikroTik hAP ac2
   ether1 (WAN) ← кабель от Huawei LAN1
   ether2 ────── switch 1G ── ПК, приставки
   ether3 ────── Archer C6 (порт LAN, режим точки доступа)
   ether4 ────── Raspberry Pi 3B (192.168.88.2)
   wlan1/wlan2 ─ Wi-Fi MikroTik (+ гостевой SSID Home-Guest)
```

## 1. Узнайте тип подключения у провайдера

Перед переводом Huawei в bridge посмотрите в его веб-интерфейсе
(обычно `192.168.100.1`, логин/пароль на наклейке или у провайдера,
часто `telecomadmin` / `admintelecom`):

- **PPPoE** — есть логин/пароль подключения. Запишите их, они понадобятся
  на MikroTik.
- **DHCP/IPoE** — адрес выдаётся автоматически.
- Также запишите **VLAN ID** WAN-подключения (в Узбекистане у GPON-провайдеров
  часто используется VLAN, например 100 или 200) — его видно в настройках
  WAN-интерфейса Huawei.

## 2. Переведите Huawei HG8546M в bridge

В веб-интерфейсе Huawei: **WAN → изменить существующее WAN-подключение**:

- Mode / Режим: `Bridge` (вместо `Route`)
- VLAN ID: оставить тот же, что был
- Binding options: привязать к `LAN1`

Если интерфейс урезан провайдером и режим Bridge недоступен — позвоните
провайдеру и попросите «перевести ONT в режим моста». Это стандартная просьба,
обычно делают удалённо за несколько минут.

⚠️ Wi-Fi на Huawei после этого отключите (если был включён) — иначе он
останется неконтролируемой дверью в интернет. В bridge-режиме обычно
отключается сам.

## 3. Переведите Archer C6 в режим точки доступа

Archer C6 больше не раздаёт свою сеть со своим DHCP — иначе устройства за ним
будут прятаться за его NAT и контролировать их по отдельности не получится.

1. Зайдите в панель Archer C6 → **Advanced → Operation Mode → Access Point**.
2. Подключите его к MikroTik кабелем **через LAN-порт** (в режиме AP это
   делается автоматически, WAN-порт становится LAN).
3. SSID/пароль оставьте прежними — устройства переподключатся сами и получат
   адреса уже от MikroTik.

## 4. Подключите всё по схеме

1. Huawei LAN1 → MikroTik ether1.
2. Switch → MikroTik ether2.
3. Archer C6 → MikroTik ether3 (или в switch — не важно, лишь бы за MikroTik).
4. Raspberry Pi → MikroTik ether4 (или в switch).

Дальше — `02-mikrotik-setup.md`.

## Проверка

- На любом устройстве: `traceroute 8.8.8.8` — первым хопом должен быть
  `192.168.88.1` (MikroTik), вторым — адрес провайдера. Если между ними
  появился `192.168.100.1` — Huawei всё ещё в режиме роутера.
- В сети не должно остаться ни одного SSID, кроме SSID MikroTik и Archer C6
  (и гостевого Home-Guest).
